    Running Commands¶

    The devbox run command executes predefined scenarios in your services. These scenarios are defined in your project configuration and provide a consistent way to run common tasks.

    Basic Usage¶

    # Run a scenario
devbox run scenario-name

# Run a scenario with additional arguments
devbox run scenario-name arg1 arg2

# Run a scenario in a specific project
devbox --name project-name run scenario-name

    Defining Scenarios¶

    Scenarios are defined in your project configuration:

    x-devbox-scenarios:
  test:
    service: api
    command: ["npm", "test"]

  migrate:
    service: db
    command: ["./migrate.sh"]
    working_dir: /app/migrations

  shell:
    service: web
    interactive: true
    tty: true
    command: ["/bin/bash"]
    user: www-data

    Scenario Options¶

    • service: Target service to run the command in
    • command: Command to execute
    • working_dir: Working directory in the container
    • entrypoint: Override container entrypoint
    • user: User to run as
    • interactive: Enable interactive mode (default: true)
    • tty: Allocate a TTY (default: true)

    Environment and Context¶

    Commands run with: - Service's environment variables - Working directory context - User permissions as specified - Network access to other services

    Examples¶

    Running Tests¶

    # Run unit tests
devbox run test

# Run specific test file
devbox run test path/to/test.js

    Database Operations¶

    # Run migrations
devbox run migrate

# Seed database
devbox run seed

    Development Tasks¶

    # Start development server
devbox run dev

# Build assets
devbox run build

    Common Issues¶

    Service Not Running¶

    • Ensure required services are started
    • Check service dependencies
    • Verify service health

    Permission Issues¶

    • Check user configuration
    • Verify volume permissions
    • Review service user settings

    Command Failures¶

    • Check command configuration
    • Verify environment variables
    • Review service logs
